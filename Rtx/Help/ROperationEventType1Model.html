Operation Event Type 1

Class Name

  • ROperationEventType1Model

Location in Objects Pane

  • Models > Model > Event > Operation Type 1

Model Description

Model Form

  • This model schedules the events of conducting an operation for different components.
    • Note: In this model, there is one operator with a capacity and there several components that form the demand for the operation.

DDM sensitivities

  • No

Properties

Object Name

  • Name of the object in Rt
  • Allowable characters are upper-case and lower-case letters, numbers, and underscore (“_”).
  • The name is unique and case-sensitive.

Display Output

  • Determines whether the model is allowed to print messages to the Output Pane.

Time Parameter

  • Time object

Operation Capacity

  • The capacity of conducting the operation (Each unit consists of material, labor, and equipment)

Operation Time List

  • List of the duration of conducting the operation for different components

Operation Finish Status List

  • List of the status parameters of the operation for different components (1 if finished, 0 otherwise)

Priority Rule

  • Method of prioritizating the components for the operation
  • The “In Order Of Input” option schedules the events based on the order of the components input by the user, on a first come first served basis
  • The “In Order Of Importance” option schedules the events based on the order of importance of components. Note that, if this option is selected, the list of the importance measure of all components must be input in the “Importance Measure List” property.

Importance Measure List

  • List of the importance measure of all components

Number Of Prerequisite Operations

  • The number of operations that have to be finished before the current operation can start (Finish-to-Start prerequisite)

Prerequisite Operations Finish Status List

  • List of the status of the prerequisite operations of the current operation for differnet components (1 if finished, 0 otherwise). The size of this list must be equal to the number of prerequisite operations times the number of the components
  • Note: First, the status parameters of the first prerequisite operation for all components are input. Next, the status parameters of the second prerequisite operation for all components are input. This procedure continues untill all of the prerequisite operations of all components are input.

Output

  • This model does not generate response objects.
  • It provides the scheduled events to the orchestrating algorithm.

Right-click Menu

Remove

  • Removes the object.
