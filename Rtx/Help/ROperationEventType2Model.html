Operation Event Type 2

Class Name

  • ROperationEventType2Model

Location in Objects Pane

  • Models > Model > Event > Operation Type 2

Model Description

Model Form

  • This model schedules the events of conducting an operation for different operators.
    • Note: In this model, there are several operators with different capacities and there is only one demand parameter.

DDM sensitivities

  • No

Properties

Object Name

  • Name of the object in Rt
  • Allowable characters are upper-case and lower-case letters, numbers, and underscore (“_”).
  • The name is unique and case-sensitive.

Display Output

  • Determines whether the model is allowed to print messages to the Output Pane.

Time Parameter

  • Time object

New Operation Demand

  • The demand for the operation, in terms of the number of the components which needs the operation.
  • This parameter must address the new demand, i.e. the demand that has just been added.

Operation Capacity List

  • List of the capacity of different operators for conducting the operation

Operation Time List

  • List of the duration of conducting the operation for different operators

Operation Finish Status List

  • List of the status parameters of the operation for different components. This parameter addresses the number of the operations that has completed by each operator.
  • Note: once the operaion is completed by an operator for \({k}\) unit of damand, the status parameter of that operator is increased by \({k}\).

Priority Rule

  • Method of prioritizating the operators for conducting the operation
  • The “In Order Of Input” option schedules the events based on the order of the operators input by the user, on a first come first served basis
  • The “In Order Of Importance” option schedules the events based on the order of importance of operators. Note that, if this option is selected, the list of the importance measure of all operators must be input in the “Importance Measure List” property.

Importance Measure List

  • List of the importance measure of all operators.

Number Of Prerequisite Operations

  • The number of operations that have to be finished before the current operation can start (Finish-to-Start prerequisite)

Prerequisite Operations Finish Status List

  • List of the status of the prerequisite operations of the current operation for differnet operators (1 if finished, 0 otherwise). The size of this list must be equal to the number of prerequisite operations times the number of the operators.
  • Note: First, the status parameters of the first prerequisite operation for all operators are input. Next, the status parameters of the second prerequisite operation for all operators are input. This procedure continues untill all of the prerequisite operations of all operators are input.

Output

  • This model does not generate response objects.
  • It provides the scheduled events to the orchestrating algorithm.

Right-click Menu

Remove

  • Removes the object.
